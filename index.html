This program is written by Ghazal Rabiei.

It uses only pure html, css and javaScript.

The space used for drawing shapes is the default canvas (a html tag provided for drawing shapes and defining coordinates).

The user can select 3 arbitrary points on the canvas. When the user selects each point, a circle is drawn using the coordinates of where was clicked.

The 3 points clicked by the user are added to an array in the same order as they are clicked.

When the user completes the selection of 3 points, the canvas shows a parallelogram that has these points as its 3 corners.

The process of creating this parallelogram is such that we should find the 4th point of the parallelogram. With these 3 points, we have 2 sides of the parallelogram (the side that passes through point 1 and point 2 and the side that passes through point 3 and point 4).

To find the 4th point, we can find the line equations of the other 2 sides and calculate their intersection and consider it as the point 4. Then we draw lines between these 4 points to see the parallelogram completely. There are some special cases that we should consider as well, that causes infinite slopes for sides.

After finding the parallelogram, we should calculate its area. Calculating the area needs the size of the height and the base. With 3 points of the parallelogram, we can calculate the area. The side that has point 3 and 4 can be considered as the base and the line that passes through point 1 and perpendicular to the base can be considered as the height and then we can calculate their sizes.

With the base and height, we can calculate the area of the parallelogram (base * height) which should be equal to the area of the circle we want to draw next.

With the area of the circle, we can easily find its radius. The center of the circle is the intersection of the diameters of the parallelogram (because this point is the center of mass of the parallelogram) and we can draw the circle based on its radius and the coordinates of the center.

To be able to move points, we can find the clicked corner of the parallelogram based on the coordinates of where the user is clicked. If the coordinates is inside the area of any points, we can clear the circle of that corner and draw its circle wherever the mouse is pointed. So, it seems like we drag the point to elsewhere. while dragging, we update the information (the coordinates the new point, the shape and area of the parallelogram and the shape of the circle).

The user can reset the canvas by clicking on the reset button, that clears the array of the points so that the user can draw the points again.
